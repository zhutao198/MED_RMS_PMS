## 五、缺陷清单

### 🔴 P0-02：`/requirements/stats` 端点未实现

| 项目 | 详情 |
|------|------|
| **等级** | 🔴 P0 |
| **现象** | GET /requirements/stats 返回 SY0101（端点不存在） |
| **影响** | 前端 RequirementList.vue 的 5 张统计卡依赖此端点（P1-3 设计提到），缺失会导致统计降级到本地聚合（可能不准确） |
| **建议修复** | 1. 后端实现 `/requirements/stats` 返回统计聚合<br>2. 或前端明确移除该端点调用 |

### 🟡 P1-02：`/requirements/1` 详情 ID=1 不存在

| 项目 | 详情 |
|------|------|
| **等级** | 🟡 P1 |
| **现象** | 默认 ID=1 的需求不存在（RQ0101 业务码） |
| **影响** | 测试时无法直接访问默认 ID；用户可能遇到 |
| **建议** | 1. 检查种子数据（init_database.sql）是否包含 ID=1 的需求<br>2. 测试时使用动态 ID |

### 🟢 P2-02：`/requirements/tree` 延迟 777ms

| 项目 | 详情 |
|------|------|
| **等级** | 🟢 P2（性能） |
| **现象** | 树查询 777ms 较高 |
| **建议** | 闭包表查询可加缓存或限制深度 |

### 🟢 P2-03：跨域端点（SOUP/任务）通过需求模块测试时 TOKEN_INVALID

| 项目 | 详情 |
|------|------|
| **等级** | 🟢 P2 |
| **现象** | `/soup`、`/project/requirements/1/tasks` 等端点返回 SY0301（TOKEN_INVALID） |
| **可能原因** | 这些端点属于其他模块（compliance / project），测试时 admin token 在这些端点上无效（**注：与模块 1 admin RBAC 失效是同一现象的不同表现**） |
