|------|-----|

| **测试类型** | 系统测试 - 端到端 |
| **优先级** | P0 |
| **需求追踪** | REQ-001, REQ-002 |

#### 测试数据

| 数据项 | 值 |
|--------|-----|
| 项目 | 心电监护仪v3.0 (PRJ001) |
| 需求层级 | URS |
| 需求标题 | ECG信号采集精度要求 |
| 优先级 | MUST |
| 法规来源 | IEC 60601-2-25 |
| 用户场景 | 临床心电监护环境 |
| 验收标准 | 采样率≥500Hz, 精度±5% |

#### 测试步骤

```
Step 1: 登录系统
  - 打开浏览器，访问 http://localhost:5173
  - 输入用户名: zhang_gong
  - 输入密码: <有效密码>
  - 点击"登录"按钮
  Expected: 登录成功，跳转至仪表盘

Step 2: 创建需求
  - 点击左侧菜单"需求管理"
  - 点击"新建需求"按钮
  - 填写表单:
    * 需求层级: URS
    * 需求标题: ECG信号采集精度要求
    * 优先级: MUST
    * 来源法规: IEC 60601-2-25
    * 用户场景: 临床心电监护环境
    * 验收标准: 采样率≥500Hz, 精度±5%
  - 点击"保存"按钮
  Expected: 需求创建成功，显示提示"需求创建成功"
  Expected: 需求编号自动生成，格式: URS-PRJ001-XXXX

Step 3: 提交评审
  - 在需求列表中找到新建需求
  - 点击"提交评审"按钮
  Expected: 弹出电子签名验证对话框
  Expected: 输入签名密码和OTP后，状态变更为"评审中"

Step 4: 验证需求状态
  - 在需求列表中查看需求状态
  Expected: 状态显示为"评审中"或"InReview"

  - 点击"提交"
  Expected: 需求状态变更为"已批准"或"Approved"

Step 6: 验证追溯链
  - 进入"追溯管理"页面
  - 查看追溯矩阵
  Expected: 新建需求出现在URS层级
```

#### 预期结果

- ✅ 需求创建成功，获得唯一编号
- ✅ 状态正确流转：DRAFT → SUBMITTED → IN_REVIEW → APPROVED
- ✅ 电子签名验证通过

### ST-REQ-002: 需求拆解流程

| 字段 | 值 |
|------|-----|
| **用例编号** | ST-REQ-002 |
| **用例标题** | 需求拆解流程 - URS拆解为PRS |
| **测试类型** | 系统测试 - 端到端 |
| **优先级** | P0 |

#### 测试数据

| 数据项 | 值 |
|--------|-----|
| 父需求 | URS-PRJ001-0001 (ECG信号采集精度要求) |
| 子需求1 | PRS-PRJ001-0001 (心电信号采样实现) |
| 子需求2 | PRS-PRJ001-0002 (信号滤波处理) |

#### 测试步骤

```
Step 1: 需求拆解准备
  - 确认URS-PRJ001-0001状态为APPROVED
  - 确认该需求未进行过拆解

Step 2: 开始拆解
  - 进入需求详情页
  - 点击"拆解为下层需求"按钮
  Expected: 跳转至拆解工作台

Step 3: 创建子需求
  - 在"子需求列表"中点击"+ 添加子需求"
  - 填写子需求1:
    * 标题: 心电信号采样实现
    * 描述: 实现500Hz采样率
    * 优先级: MUST
  - 再次点击"+ 添加子需求"
  - 填写子需求2:
    * 标题: 信号滤波处理
    * 描述: 实现低通滤波算法
    * 优先级: SHOULD
  - 点击"保存拆解"按钮
  Expected: 子需求创建成功

Step 4: 建立追溯链接
  - 进入"追溯管理"
  - 创建追溯链接:
    * 源: URS-PRJ001-0001
    * 目标: PRS-PRJ001-0001
    * 类型: satisfies
  - 创建追溯链接:
    * 源: URS-PRJ001-0001
    * 目标: PRS-PRJ001-0002
    * 类型: satisfies

Step 5: 标记拆解完成
  - 返回需求管理
  - 将父需求状态改为"已拆解"
  Expected: 状态变更为Decomposed
```

#### 预期结果

- ✅ URS成功拆解为2个PRS
- ✅ 追溯链接自动建立（包含反向链接）
- ✅ 闭包表正确维护父子关系
- ✅ 状态变更为DECOMPOSED

