# OA 浏览器抓取指南 — R101 方案 B 变体

## 📋 适用场景

- 不想麻烦 OA 管理员配置集成
- 不想触碰泛微 e-cology v8 的 SOAP/RSA 加密登录
- 你的账号已经在浏览器登录 OA
- 接受"手动在浏览器跑脚本触发同步"的工作流

## 🎯 核心思路

**跳过**:
- ❌ 泛微登录(RSA + AES 加密)
- ❌ SOAP over HTTP 协议
- ❌ OA 后台集成配置

**利用**:
- ✅ 你已经登录的浏览器 session(cookie 自动带)
- ✅ SPA 已加载的 DOM 数据
- ✅ SPA 内部 API(/api/workflow/wo/...)
- ✅ Med-RMS 后端接收端点

## 📁 文件

- `tools/oa_browser_capture.js` — 浏览器抓取脚本(粘贴到 Chrome Console 运行)
- 后端接收端点 `OaSyncController`(已部署)
  - `POST /api/oa-sync/subcompanies` — 分部
  - `POST /api/oa-sync/departments` — 部门
  - `POST /api/oa-sync/users` — 人员
  - `POST /api/oa-sync/urgent-requirements` — 紧急需求单
  - `POST /api/oa-sync/requirement-cards` — 需求采集卡
  - `GET /api/oa-sync/status` — 健康检查

## 🚀 使用步骤

### Step 1: 登录 OA
浏览器打开 `http://192.168.10.64:81`,登录(你的工号 000296)

### Step 2: 打开 DevTools
按 `F12` → 切到 **Console** 标签

### Step 3: 粘贴脚本
复制 `tools/oa_browser_capture.js` 的全部内容,粘贴到 Console,回车执行

### Step 4: 看输出
脚本会按顺序:
1. 抓取组织架构 → POST `/oa-sync/subcompanies`
2. 抓取人员 → POST `/oa-sync/users`
3. 抓取紧急需求单 → POST `/oa-sync/urgent-requirements`
4. 抓取需求采集卡 → POST `/oa-sync/requirement-cards`

每步打印 HTTP 状态 + 返回 JSON

## ⚠️ 关键依赖

| 项 | 状态 |
|---|---|
| 浏览器已登录 OA | ✅ 你已登录 |
| Med-RMS 后端在 localhost:8080 | ✅ 已部署 |
| 你要导航到对应页面 | ⚠️ **脚本只在 SPA 渲染后才有数据** |

## 🎯 推荐操作流程

1. **同步组织架构**:导航到 OA 左侧菜单 `组织架构 → 分部管理`(或通讯录),F12 粘贴脚本,等"✅"出现
2. **同步人员**:导航到 `通讯录`,F12 粘贴脚本
3. **同步紧急需求单**:导航到 `流程 → 我的请求`(或工作流列表),筛选 535,F12 粘贴脚本
4. **同步需求采集卡**:同上,筛选 554,F12 粘贴脚本

## 🐛 已知限制

- **数据量限制**:DOM 抓取受限于当前页面渲染(可能只显示第一页)
- **字段映射**:OA 字段(_fullname / _code) → Med-RMS 字段(name / code)已做映射,但不保证 100% 准确
- **用户关联部门**:dept_id 自动通过 departmentname 反查,**重名时会错配**(后续可优化)
- **CORS**:脚本默认从 `localhost:8080` 拉数据,如有跨域会失败(已在后端 `WebConfig.corsFilter` 放过 `localhost:*`)

## 🔧 修复记录

### R101b → R101h 累计修改

1. **`OaSyncController.java`** 复制到 med-rms-web 模块(避免跨 jar ComponentScan 限制)
2. **`@RequestMapping` 从 `/api/oa-sync` 改为 `/oa-sync`**(因 `server.servlet.context-path=/api` 已加 `/api` 前缀)
3. **`User` 创建时设默认 BCrypt `passwordHash`**(因 t_user.password_hash NOT NULL)
4. **application.yml 暴露 `actuator/mappings` 端点**(调试用,可保留)
5. **sed 修复 MedRmsApplication** 移除 `@Import`(已不需要)

- ❌ ServerContextPath 重复导致路径不匹配 → 移除 `/api` 前缀
- ❌ t_user.password_hash NOT NULL → 显式设默认值
