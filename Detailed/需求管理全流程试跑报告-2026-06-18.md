## 发现的问题(综合)

### 问题1: `/review` 端点无 Submitted 中间状态 ⚠️
- **现象**:`POST /requirements/{id}/review?reviewerId=1` 状态直接 ReviewApproved
- **影响**: 状态机合并了 Submit + Approve 两步,前端没有独立触发 Submit 按钮(只有"提交评审"按钮一次到 Approved)
- **严重度**: 低(可能是设计简化)

### 问题2: `/review-multi` 端点入参命名 ⚠️
- **现象**:`?reviewers=1,2` 报"缺少必要参数: reviewerId"
- **实际**: 后端期望单个 `reviewerId`,非数组
- **严重度**: 中(多人评审 API 契约不清,前端可能没用此端点)

### 问题3: 基线 `/baseline-requirements` 端点入参 ⚠️
- **现象**: body 含 `baselineId/requirementId` 报"缺少必要参数: baselineId"
- **可能**: 端点期望 query 参数而非 body
- **严重度**: 中(基线添加需求流程可能受影响)

### 问题4: 拆解后 `/children` 端点返 None ⚠️
- **现象**:`GET /requirements/{srs_id}/children` 返 data=null
- **替代**:`/tree?projectId=1` 仍能查到(317 个 root)
- **严重度**: 中(API 契约问题,前端可能有兜底)

### 问题5: 追溯图谱 SVG 不渲染 🆕
- **现象**: `/traceability/graph` 页面加载但 svg_count=0
- **可能**: ECharts 加载失败 / 空数据状态
- **严重度**: 中(图谱页功能不可用,影响 5.追溯流的 UI 验证)
- **建议**: 检查前端 console + ECharts 初始化逻辑
