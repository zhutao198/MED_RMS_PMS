## 交付物使用指南

### 工程师编码入口

1. **先读数据字典** → 了解所有枚举值、状态码、编码规则
2. **再读对应模块详细设计** → 类图+时序图+伪代码，可直接编写 Java 代码
3. **执行 DDL 建库** → `psql -f 02-DDL/med_rms_ddl.sql`
4. **导入 Swagger** → 将 YAML 导入 Swagger Editor/Postman，生成 API 客户端
5. **参考原型** → 浏览器打开 HTML 原型，确认交互逻辑

### 文档依赖关系

```
数据字典 ← 所有详细设计文档引用
    ↑
详细设计(类图/时序图/伪代码) ← 编码的直接依据
    ↑
DDL ← 详细设计中定义的实体类对应建表
    ↑
OpenAPI ← 详细设计中定义的 Service 接口对应 API
    ↑
权限设计 ← 编码时集成 Spring Security
    ↑
异常容错 ← 编码时集成全局异常处理
    ↑
日志审计 ← 编码时集成 AOP 审计切面
```

## 模块实现优先级（MoSCoW）

| 优先级 | 模块 | 理由 |
|--------|------|------|
| MUST | req-mgr（需求管理） | 核心业务，4层需求+CTI+闭包表 |
| MUST | trace-mgr（追溯管理） | 合规刚需，追溯矩阵+覆盖率 |
| MUST | compliance（合规管理） | 21 CFR Part 11 + IEC 62304 |
| MUST | e-sign（电子签名） | 签名密码+OTP，Part 11 合规 |
| MUST | sys-mgr（系统管理） | 基础设施，用户/角色/权限 |
| SHOULD | chg-mgr（变更管理） | 变更控制，ISO 13485 |
| SHOULD | risk-mgr（风险管理） | ISO 14971 风险管理 |
| SHOULD | proj-mgr（项目管理） | IPD/DCP 门控 |
| COULD | report（报表仪表盘） | CQRS Lite，可后置 |

## 建议编码顺序

1. **Phase 1：基础设施**（1-2周）
   - DDL 建库建表
   - sys-mgr（用户/角色/权限/JWT）
   - 全局异常处理 + 审计日志 AOP

2. **Phase 2：核心业务**（3-4周）
   - req-mgr（需求管理 + CTI + 闭包表）
   - e-sign（电子签名 + OTP）
   - trace-mgr（追溯管理）

3. **Phase 3：合规扩展**（2-3周）
   - compliance（审计日志/SOUP/基线/问题报告/IEC62304）
   - chg-mgr（变更管理）

4. **Phase 4：支撑功能**（1-2周）
   - risk-mgr + proj-mgr + report

5. **Phase 5：前端集成**（贯穿）
   - Vue3 + Element Plus 页面开发
   - 参考交互原型实现
