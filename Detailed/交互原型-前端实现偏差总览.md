## 五、根因分析

### 5.1 后端枚举与原型术语未对齐
- `TraceGap.type` 字段值（MISSING_CHILDREN/ORPHAN/NO_TEST_CASE） vs 原型（unlinked/suspect）
- `RiskItem.severity` 字段值（CRITICAL/MAJOR/MINOR/NEGLIGIBLE） vs 原型（1-4 数字）
- 严重度大小写（critical vs CRITICAL）
- 法规类型（nmpa/fda/ce/iec vs IEC62304/ISO14971/21CFR11/GMP）
- SOUP 风险等级少 critical 枚举
- SOUP 组件类型维度（"来源" vs "用途"）

**根因**：详细设计 v1.1 未明确枚举值，前端自由发挥，后端按自由发挥的命名落地。

### 5.2 多个独立页面未建（路由/组件双重缺失）
- 审计域：2 路由 + 2 vue 全部 0%
- 签名域：5 路由缺失
- 合规域：9 独立页缺失
- 项目域：6 路由缺失
- 系统域：4 路由缺失

**根因**：75 个原型在概要设计 v1.2 已批准，但前端按"按需实现"原则跳过了部分 COULD 优先级页面（虽然缺失清单 v1.5 报告 100% 全覆盖，但覆盖度 ≠ 实现度）。

### 5.3 关键合规字段在前端未实现
- `signatureValue`、`auditHash`、`chainValid` 在前端 0 命中
- 签名意图、签名含义、IP、设备、验证状态 5 列缺失
- DCP 5 门控无 signUser 绑定

**根因**：21 CFR Part 11 哈希链算法在审计 Aspect 中已实现，但前端未消费后端的合规字段。

### 5.4 编辑页未对接 API
- `ReqEdit.vue` 纯 Mock，写死 formData，handleSave 弹 toast 即结束
- 缺口集中在"详情-编辑-保存"流程，前端无 update 调用

**根因**：需求域"详情-编辑"流程后端 API 完善，前端页面骨架先行但未对接。
